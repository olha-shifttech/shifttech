## 3. Проблема



Більшість клієнтів не знає:



- код своєї коробки передач;

- виробника коробки;

- точну модифікацію;

- внутрішні технічні назви;

- різницю між АКПП, CVT, DSG та іншими типами коробок.



Натомість клієнт майже завжди знає:



- марку автомобіля;

- модель;

- рік;

- двигун;

- симптом, який він відчуває.



Тому сайт не може вимагати від клієнта знання коробки передач на вході.

## 4. Головний принцип



ShiftTech використовує дві навігаційні логіки одночасно:



1. UX Navigation — навігація для людини.

2. Knowledge Graph — архітектура знань для системи та AI.



Ці два шари не конфліктують. Вони доповнюють один одного.

## 7. Чому саме так



Людина мислить автомобілем.



AI і база знань мислять зв'язками між сутностями.



Тому:



UX не дорівнює архітектурі даних.



Для людини потрібно зробити простий шлях входу.



Для системи потрібно побудувати правильний граф знань.

## 8. Головне правило



Для користувача:



Автомобіль → Двигун → Коробка → Проблема → Рішення



Для системи:



Коробка → Автомобілі → Симптоми → Помилки → Кейси → Деталі



Обидві моделі повинні працювати одночасно.

## 11. Заборонено



Заборонено будувати платформу тільки навколо автомобілів.



Це створить дублікати, тому що одна коробка може стояти на багатьох авто.



Заборонено будувати платформу тільки навколо коробок.



Це буде незручно для клієнтів, бо більшість людей не знає код коробки.



Правильний варіант — подвійна навігація.

---



## 14. Наслідки для AI



AI повинен розуміти:



- клієнт може не знати коробку;

- запит по авто має бути пов'язаний з коробкою;

- запит по симптому має бути пов'язаний з авто і коробкою;

- запит по помилці має бути пов'язаний з коробкою, авто і кейсами.



AI не повинен вимагати від клієнта технічних знань, якщо їх можна визначити через авто, двигун, VIN або симптом.



---



## 15. Наслідки для бази знань



Кожна сутність повинна мати зв'язки:



Автомобіль пов'язаний з:



- коробками;

- двигунами;

- симптомами;

- помилками;

- кейсами.



Коробка пов'язана з:



- автомобілями;

- симптомами;

- помилками;

- запчастинами;

- ремонтами;

- кейсами.



Симптом пов'язаний з:



- автомобілями;

- коробками;

- помилками;

- кейсами;

- діагностикою.



---



## 16. Критерій правильності



Нова сторінка або новий матеріал вважається правильно спроєктованим, якщо:



1. Людина може знайти його через автомобіль.

2. AI може знайти його через коробку передач.

3. Матеріал має зв'язки з іншими сутностями.

4. Інформація не дублюється.

5. Є зрозумілий наступний крок для клієнта.
